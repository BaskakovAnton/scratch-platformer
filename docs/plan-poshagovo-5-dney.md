# Пошаговый план на 5 дней (только файлы из `DZ/`)

Учитель — папа, ученик — дочь.  
Общий план и карточки: `docs/plan-nedelya-kodlox.md`.  
Здесь — **что открыть и какие блоки трогать**.

### Правило перед каждым днём

1. Scratch → **Файл → Загрузить с компьютера** → файл из `DZ/`.  
2. Сразу **Файл → Сохранить на компьютер** под новым именем (копия).  
3. Оригинал в `DZ/` не перезаписывать.  
4. В LMS сдавать скрин **копии**.

# День 1 · Понедельник — Iron Man

## Файлы

- Открыть: `DZ/Iron Man заготовка.sb3`  
- Сохранить как: `DZ/work/ironman-yana.sb3` (папку `work` можно создать)

## Как устроена игра (покажи дочери)

1. Зелёный флаг → видна **Кнопка**.  
2. Клик по кнопке → сообщение **«Старт»**.  
3. Летает **Железный человек**, справа летят **Ракеты**.  
4. Касание ракеты → **game over**, фон **Поражение**.  
5. Очки растут сами раз в секунду.

## Шаг A — поиграть (3 мин)

1. Зелёный флаг.  
2. Нажать кнопку на сцене.  
3. Спросить: «Когда проигрыш? Что такое очки?»

## Шаг B — ДЗ: доработать (20–30 мин)

Делайте **1–2 улучшения** на выбор (школе нужен скрин доработки):

### Вариант 1 — герой красивее

1. Спрайт **Железный человек** → вкладка **Костюмы**.  
2. Можно подправить размер: в скриптах после «Старт» добавить  
   `установить размер 80 %` (или 60–100 — на глаз).  
3. Зелёный флаг → кнопка → проверить.

### Вариант 2 — ракеты реже / чаще (сложность)

1. Спрайт **Ракета**.  
2. Найти скрипт:  
   `когда я получу [Старт]` → `повторять всегда` → `создать клон` → **`ждать (случайное от 1 до 5)`**.  
3. Изменить на проще для дочери:  
   `ждать (случайное от 2 до 5)`  
   или сложнее: `от 1 до 3`.  
4. Проверить игрой.

### Вариант 3 — ракеты медленнее

1. Спрайт **Ракета**, скрипт клона с  
   `плыть 4 секунд в x: -200 y: …`  
2. Поставить **5** или **6** секунд — медленнее.  
3. Проверить.

### Вариант 4 — старт героя

1. Спрайт **Железный человек**.  
2. Блок `перейти в x: (-170) y: (6)` — можно чуть сдвинуть y, если герой обрезается.

## Шаг C — сдача

1. Зелёный флаг → старт → сыграть → сделать скрин.  
2. LMS → ДЗ урока 1 → прикрепить скрин.

# День 2 · Вторник — Капитан Взлёт

## Файлы

- Открыть: `DZ/Капитан Взлёт.sb3`  
  *(или свой `busya.sb3` — те же шаги, имена спрайтов другие)*  
- Сохранить как: `DZ/work/kapitan-yana.sb3`

## Как устроена игра

| Спрайт | Роль |
|--------|------|
| Спрайт супергероя | пробел = вверх, иначе падает (`СкоростьY`) |
| Препятствия | клоны едут влево, `ждать 2`, `x −5` |
| Игра окончена | экран конца |
| Sprite1 | монетки (доп.) |

Управление: **пробел**.

## Шаг A — поиграть (3 мин)

Зелёный флаг, полетать, врезаться. Показать переменную **Счёт**.

## Шаг B — Базовый ДЗ: вид (15–20 мин)

### Фон

1. Сцена → **Фоны** → загрузить свой фон **или** выбрать из галереи.  
2. Можно оставить школьный «Фон».

### Герой

1. Спрайт **Спрайт супергероя** → **Костюмы** → загрузить новый / Бусю.  
2. В коде уже есть `установить размер 40 %` — подогнать **30–50**, чтобы не касался труб сразу.

### Трубы (препятствия)

1. Спрайт **Препятствия** → **Костюмы** → другой рисунок (или когтеточка из `img/world/`).  
2. Центр костюма roughly в проёме (крестик в редакторе).  
3. Зелёный флаг — проверить полёт и стол.

## Шаг C — Средний ДЗ: сложность (15 мин)

Спрайт **Препятствия**, скрипт зелёного флага:

```
когда флаг нажат
задать Счёт = 0
спрятаться
повторять всегда
  задать [высота прохода] = случайное от (-90) до (90)
  создать клон себя
  передать [Создатьмонету]
  ждать (2) секунды
```

И в клоне: `изменить x на (-5)`.

**Что крутить с дочерью:**

| Стало | Блок | Было → стало (пример) |
|-------|------|------------------------|
| Реже трубы | `ждать (2)` | `2` → `2.5` или `3` |
| Чаще | `ждать` | `2` → `1.5` |
| Медленнее | `изменить x на (-5)` | `-5` → `-3` или `-4` |
| Быстрее | `изменить x` | `-5` → `-6` |

После каждого изменения — прогон 30 сек. Оставить числа, которые ей нравятся.

## Шаг D — Уверенный ДЗ: обложка (15 мин)

1. Сцена → новый фон **или** новый спрайт «Обложка».  
2. Написать название (инструмент Текст в костюмах / внешний рисунок).  
3. Скрин обложки → сдача «уверенный» в LMS.  
4. Для игры обложку можно спрятать по клику (как кнопка в Iron Man) — по желанию.

## Сдача

Три скрина/файла: базовый, средний, уверенный.

# День 3 · Среда — Кликер (собираем из PNG)

> В `DZ/` **нет** готового `.sb3` кликера — только картинки.  
> Собираем мини-проект с нуля за один день + ДЗ.

## Файлы картинок

Папка: `DZ/Заготовка-20260720T073938Z-1-001/Заготовка/`

- `Фон.png`  
- `Главный герой.png`  
- `50 монет (серая).png` / `50 монет (коричневая).png`  
- `100 монет (серая).png` / `100 монет (коричневая).png`

Сохранить проект как: `DZ/work/kliker-yana.sb3`

## Шаг A — собрать игру (25–30 мин)

### 1) Сцена

1. Новый проект Scratch (или удалить кота).  
2. Сцена → Фон → загрузить `Фон.png`.

### 2) Переменные (для всех спрайтов)

Создать:

- `Монеты`  
- `монеты за клик` (начать с **1**)  
- `монеты в секунду` (начать с **0**)

Показать галочками на сцене.

### 3) Герой

1. Загрузить спрайт `Главный герой.png`.  
2. Имя: **Герой**.  
3. Если чёрный квадрат — ластиком убрать фон в костюме.  
4. Размер подогнать (~40–70%).

Код героя:

```
когда флаг нажат
  задать [Монеты] = 0
  задать [монеты за клик] = 1
  задать [монеты в секунду] = 0

когда этот спрайт нажат
  изменить [Монеты] на (монеты за клик)
```

### 4) Пассивный доход (сцена или герой)

```
когда флаг нажат
  повторять всегда
    ждать (1) секунду
    изменить [Монеты] на (монеты в секунду)
```

### 5) Кнопка «50 монет» (двойной клик)

1. Спрайт из `50 монет (серая).png`, имя **Кнопка50**.  
2. Второй костюм: `50 монет (коричневая).png`.  
3. Код:

```
когда этот спрайт нажат
  если <(Монеты) ≥ (50)> то
    изменить [Монеты] на (-50)
    изменить [монеты за клик] на (1)
    следующий костюм
    ждать (0.2) секунды
    переключить костюм на [серая]
```

*(«двойной клик» в школьном смысле = усиление клика; достаточно +1 к «монеты за клик».)*

### 6) Кнопка «100 монет»

Аналогично, спрайт **Кнопка100**, костюмы серая/коричневая:

```
когда этот спрайт нажат
  если <(Монеты) ≥ (100)> то
    изменить [Монеты] на (-100)
    изменить [монеты в секунду] на (1)
    … мигание костюма …
```

Проверка: клики копят монеты → покупки работают.

## Шаг B — ДЗ базовый

1. Уже стоит школьный фон — можно сменить на другой из галереи.  
2. Изменить **размер** героя (ползунок или `установить размер`).  
3. Скрин.

## Шаг C — ДЗ средний

На **Кнопка50** или **Кнопка100** в скрипт клика добавить:

```
включить звук [Pop]   // любой короткий из библиотеки
```

Проверить ухом.

## Шаг D — ДЗ уверенный — бонус

1. Новый спрайт (звезда из галереи или нарисовать). Имя **Бонус**.  
2. Код:

```
когда флаг нажат
  показаться
  перейти в x: (100) y: (80)   // любое место

когда этот спрайт нажат
  изменить [Монеты] на (10)    // она выбирает 5/10/20
  спрятаться
```

3. Скрин: до клика / после рост монет.

## Сдача

Базовый + средний + уверенный в LMS (урок 1-3).

# День 4 · Четверг — Лабубу ч.1

## Файлы

- Открыть: `DZ/лабубу/лабубу/Испытание Лабубу.sb3`  
- Сохранить как: `DZ/work/labubu-ch1-yana.sb3`  
- Картинки (по желанию): `DZ/лабубу/лабубу/Материалы для оформления игры/`

**Не открывать:** `лабубу.sb3` (пустой).

## Как устроена игра

- Стрелки: вправо/влево/вверх/вниз, шаг **5**.  
- Стена = цвет `#010101` (почти чёрный): если коснулся — откат шага.  
- 4 звезды: касание Лабубу → `Звёзды +1`, звук, спрятаться.  
- Касание **Финиш** → сказать «Победа!» → стоп.  
- Фон: есть вечный звук `Dance Head Nod` — можно **отключить** скрипт со звуком, чтобы не мешал (удалить или отсоединить).

Старт героя: `x=-20 y=159`. Финиш: `x=22 y=-164`.

## Шаг A — поиграть (5 мин)

Пройти лабиринт, собрать звёзды, дойти до финиша.  
Спросить: «Что нельзя? Что собирать? Где победа?»

## Шаг B — Базовый ДЗ: видно и красиво (20 мин)

1. Сцена → Фоны — сменить фон (галерея) **или** оставить.  
2. Спрайт **Спрайт Лабубу** → Костюмы → загрузить  
   `Материалы…/Спрайт Лабубу.png` (ластиком убрать чёрный фон).  
3. Размер героя подогнать, чтобы проходил коридоры.  
4. Спрайт **Финиш** — можно поменять костюм на флажок из галереи.  
5. Проверка: герой, стены, звёзды, финиш **хорошо видны**.  
6. Пройти до «Победа!».

> Стены сейчас — костюм «Референс лабиринта». Если меняете фон сцены, стены должны остаться контрастными (тёмные линии на светлом или наоборот).

## Шаг C — Средний ДЗ: ещё бонус (20–25 мин)

1. Дублировать одну **Звезду** (правый клик по спрайту → дублировать)  
   **или** новый спрайт из галереи «Ball» / сердце. Имя: **Бонус1**.  
2. Скопировать код звезды (тот же смысл):

```
когда флаг нажат
  задать [Звёзды] = 0     // лучше НЕ сбрасывать здесь, если уже сбрасывает Звезда1
  перейти в x: (…) y: (…)  // НОВОЕ место в лабиринте
  показаться
  повторять всегда
    если <касается [Спрайт Лабубу]?> то
      изменить [Звёзды] на (1)
      включить звук […]
      спрятаться
```

**Важно:** сброс `Звёзды = 0` должен быть **только в одном** месте (например только у Звезда1). У новых бонусов строку сброса **удалить**, иначе счёт обнуляется.

3. Поставить 2–3 бонуса в разных местах (дубликаты с разными x,y).  
4. Пройти: собрать новые + старые звёзды + финиш.

## Шаг D — Уверенный ДЗ: один стиль (15 мин)

Выбрать тему (лес / космос / сладкий…).  
Подогнать: фон сцены + костюм Лабубу + вид финиша (+ бонусы).  
Скрин «всё в одном мире».

## Сдача

Три уровня урока 1-4.

# День 5 · Пятница — Лабубу ч.2 (достраиваем на ч.1)

> и добавляем уровень 2 + ловушки + экраны из папки материалов.

Картинки:

- `Лабиринт2.png`  
- `Враг.png`  
- `Ловушка.png`  
- `Экран победы.png`  
- `Экран окончания игры.png`

## Шаг A — идея для дочери (3 мин)

«Вчера один этаж. Сегодня два. На втором — ловушки. Потом картинка Победа.»

## Шаг B — Базовый ДЗ: тест + второй уровень (30–40 мин)

### B1. Подготовить костюмы стен

1. Спрайт **Стены**.  
2. Костюм 1 — текущий лабиринт (уровень 1).  
3. Загрузить `Лабиринт2.png` как **костюм 2** (имя: `лабиринт2`).  
4. Если линии лабиринта не чёрные `#010101`, касание цвета может сломаться — тогда:  
   - либо дорисовать стены чёрным в редакторе,  
   - либо временно оставить один лабиринт и сделать «уровень 2» = тот же макет + ловушки (честно сказать в сдаче).

### B2. Сообщения

Создать сообщения (жёлтые блоки):

- `Уровень 2`  
- `Победа`  
- `Проигрыш`

### B3. Переход на уровень 2

Сейчас победа = касание **Финиш**. Меняем логику у **Спрайт Лабубу**:

**Было:** касание Финиш → «Победа!» → стоп.  

**Стало (уровень 1):**

```
когда флаг нажат
  задать [номер уровня] = 1    // создать переменную
  …
  повторять всегда
    если <<касается [Финиш]?> и <(номер уровня) = (1)>> то
      передать [Уровень 2]
```

Спрайт **Стены** (или Сцена):

```
когда я получу [Уровень 2]
  переключить костюм на [лабиринт2]
```

Спрайт **Спрайт Лабубу**:

```
когда я получу [Уровень 2]
  задать [номер уровня] = 2
  перейти в x: (…) y: (…)   // старт второго лабиринта (подобрать игрой)
  // показать снова звёзды уровня 2 — по желанию дубликаты
```

Спрайт **Финиш** — на уровне 2 переставить координаты (куда выход на `Лабиринт2`).

**Победа только на уровне 2:**

```
если <<касается [Финиш]?> и <(номер уровня) = (2)>> то
  передать [Победа]
```

### B4. Экран победы

1. Спрайт из `Экран победы.png`, имя **ЭкранПобеды**.  
2. Код:

```
когда флаг нажат
  спрятаться

когда я получу [Победа]
  показаться
  идти на передний план
  остановить все
```

### B5. Чеклист тестировщика (базовый LMS)

Вместе пройти и отметить:

1. Старт уровня 1  
2. Финиш → переход на уровень 2 (сменились стены)  
3. Пройти уровень 2  
4. Финиш → экран победы  

Баги чинить по одному.

## Шаг C — Средний ДЗ: звуки + ловушка/враг (20 мин)

1. Спрайты из `Ловушка.png` и/или `Враг.png`.  
2. Поставить на **уровне 2** (или обоих).  
3. У Лабубу:

```
повторять всегда
  если <касается [Ловушка]?> то
    включить звук [Oops]      // любой «плохой»
    передать [Проигрыш]
```

4. Спрайт **ЭкранКонца** из `Экран окончания игры.png`:

```
когда флаг нажат
  спрятаться
когда я получу [Проигрыш]
  показаться
  остановить все
```

5. На победу:

```
когда я получу [Победа]
  включить звук [Cheer] / играть до конца
  …
```

Проверить специально: врезаться в ловушку; выиграть честно.

## Шаг D — Уверенный ДЗ: стартовый экран (15–20 мин)

1. Спрайт **Меню** (нарисовать или коллаж): название +  
   «Стрелки — ходить · Звёзды — собирать · Ловушки — нельзя · Флаг — начать».  
2. Код:

```
когда флаг нажат
  показаться          // Меню
  // Лабубу, ловушки — спрятать

когда клавиша [пробел] нажата   // или клик по Меню
  спрятаться          // Меню
  передать [СтартИгры]
```

3. Вся игровая логика Лабубу/звёзд слушает **СтартИгры** вместо голого «когда флаг»  
   *(минимальный вариант: по пробелу спрятать меню и не трогать остальное — флаг уже запустил игру под меню; тогда меню просто перекрывает экран до пробела).*

Простой рабочий минимум для сдачи:

- Меню поверх всего при флаге.  
- По пробелу: меню спрятать.  
- Игра уже бежит под ним.

Скрин меню + название.

Базовый / средний / уверенный урока 1-5.
